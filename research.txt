# 🎯 PROJECT OBJECTIVE

Build a fully modular ML system that:

- Detects malicious vs benign NPM packages
- Processes large-scale datasets (10K–100K+ packages)
- Extracts static + metadata + security features
- Uses an optimized pipeline architecture (no redundancy)
- Trains multiple ML models
- Uses ensemble/fusion methods for final prediction
- Supports fast inference and scalable design

---

# 📦 INPUT DATA

Raw NPM package dataset containing:

- package.json
- JavaScript source files
- dependency trees
- install scripts (preinstall/postinstall)

Labels:
- 0 → benign package
- 1 → malicious package

---

# ⚙️ SYSTEM DESIGN REQUIREMENTS

## 1. CLEAN PIPELINE ARCHITECTURE

The system MUST follow a strict modular pipeline:

### Stage 1: Feature Extraction
- Extract features once only (no repeated scanning)
- Scan only relevant files (.js, package.json, scripts)
- Extract security-related signals:
  - dependency count
  - install scripts presence
  - suspicious API usage (eval, exec, child_process, fs, net, http)
  - file structure complexity
  - lightweight entropy / obfuscation indicators

### Stage 2: Dataset Builder
- Convert extracted features into structured tabular dataset
- Store dataset in optimized format (Parquet preferred)
- Support incremental batch processing

### Stage 3: Preprocessing
- Handle missing values
- Normalize numeric features if required
- Remove constant/low-variance features
- Ensure reproducibility

---

# 🤖 2. MULTI-MODEL TRAINING SYSTEM

Train multiple independent models:

### Required models:
- LightGBM
- XGBoost
- Random Forest
- Logistic Regression (baseline)

Each model must:
- Be trained independently
- Be evaluated separately
- Output probability predictions
- Be saved individually

Metrics required:
- precision
- recall
- F1-score
- confusion matrix

---

# 🔥 3. ENSEMBLE / FUSION SYSTEM (CRITICAL)

Implement a fusion layer that combines all model outputs.

## REQUIRED METHODS:

### Option A: Soft Voting (Preferred)
- Average probability outputs from all models
- Final prediction = weighted or equal average

### Option B: Weighted Voting
- Assign weights based on model performance

Example:
- LightGBM = 0.4
- XGBoost = 0.3
- Random Forest = 0.2
- Logistic Regression = 0.1

### Option C: Stacking (Advanced)
- Train a meta-model using predictions of base models
- Meta-model outputs final classification

---

# ⚡ 4. PERFORMANCE REQUIREMENTS

- Feature extraction must run only once (cached)
- No redundant computation in training
- Must support batch processing for large datasets
- Must scale to 100K+ packages efficiently
- Must use multiprocessing where possible
- Must minimize I/O overhead

---

# 🔮 5. INFERENCE SYSTEM

The system must:

- Load all trained models
- Generate predictions from each model
- Apply ensemble fusion logic
- Output:
  - final classification (benign/malicious)
  - probability score
  - model-wise confidence breakdown

Must be optimized for fast inference.
